How do I set up port forwarding on the router?

To set up port forwarding, you must be logged into the router's user interface. Some routers also require the "extended view" to enter port forwarding.

In the following we have described how to create a port forwarding for the most common routers. Please note that the description may vary slightly. In this case, read the instructions of your router or check the FAQs of the router manufacturer.

Fritzbox:

  • Select „Erweiterte Ansicht“
  • Menu Internet > Shares > tab "Portfreigaben"
  • Click on the button „Gerät für Freigaben hinzufügen“
  • Select eBlocker
  • Click on the button „Neue Freigabe“
  • Click on port sharing
  • Select „Andere Anwendung“ in the „Anwendung“ tab
  • Choose a name (for example: eBlocker Mobile)
  • Protocol = UDP
  • Port to device = 1194
  • External port required = from 1194 to 1194
  • Click on „Freigabe aktivieren“
  • and click on „OK“


TP link:

  • Menu Forwarding > Virtual Servers
  • Button „Add New... “
  • Service port = 1194
  • Internal Port = 1194
  • IP Address = the IP address of your eBlocker
  • Protocol = All
  • Status = Enabled
  • Click the „Save“ button

TP link (alternative version):

  • Advanced menu > NAT Forwarding > Port-Triggering
  • Button „(+) Add“
  • Applikation = freely assigned (for example: eBlocker Mobile)
  • Triggering Port = 1194
  • Triggering Protokoll = UDP
  • External Port = 1194
  • External Protokoll = UDP
  • Enable ths entry = (set checkmark)
  • Click on the „OK“ button

 

Netgear:

  • Select the "Advanced" tab
  • Menu Advanced Setup > Port Forwarding/Port Triggering > click on the „Add Custom Service“ Button
  • Assign service name freely (for example: eBlocker Mobile)
  • Protocol = UDP
  • External starting port = 1194
  • External Ending Port = 1194
  • Check „use the same portrange for the internet port“
  • eBlocker Enter IP address or select the eBlocker from the list of devices
  • Click on the „Apply“ button


D-Link:

  • Select the „Advanced“ tab
  • Select „Virtual Servers“ from the menu
  • Activate the first unused virtual server
  • Now assign a name (for example: eBlocker Mobile)
  • Public= 1194 to 1194
  • Protocol = UDP
  • Enter the IP address of your eBlocker at „Virtual Servers“
  • Private = 1194 to 1194
  • Click on the „Save settings“ button

Apple Airport:

  • Start the macOS Airport Utility
  • Select the tab "Network"
  • Click the "+" button at "Port Settings"
  • Add a name like "eBlocker Mobile" in the filed "Description"
  • Use Public TCP and UCP port 1194
  • Input your eBlockers IP-Address in the "Private IP Address" field
  • Use Private TCP and UCP port 1194
  • Click on the button "Save"
  • Click on the button "Update"

Asus Router:

  • Start the Asus Merlin Software
  • Select "Advanced Settings" > "WAN" > "Port Trigger" tab
  • Click the "yes" radio button
  • "Description": Enter a short name e.g. eBlocker
  • insert 1194 in the field "Trigger Port" and also 1194 in the field "Incoming Port"
  • choose UDP in both "Protocol" dropdown menus
  • Click "Apply"

Speedport W724V

  • Tab „Internet“ > Menu „Port activation“
  • click on the button „Create other UDP redirection“
  • insert 1194 in the fields
  • choose the eBlocker or his IP-Address from the Dropdownfeld
  • and click on the „SAVE“ button
